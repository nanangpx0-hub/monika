# QUICK REFERENCE - PETUGAS PENGOLAHAN
*Sistem MONIKA*

## 🔑 AKSES PROCESSING
✅ Dashboard (view only)  
✅ Lihat semua dokumen  
✅ Mark entry dokumen  
✅ Lapor anomali  
❌ Setor dokumen  

## 📊 DASHBOARD
- Total dokumen perlu diproses
- Dokumen sudah selesai
- Dokumen error perlu tindak lanjut

## 🎯 TUGAS UTAMA

### 1. Proses Dokumen "Uploaded"
**Menu:** Dokumen Survei
- Fokus pada status "Uploaded"
- Review dokumen fisik/digital
- Buat keputusan: Valid atau Error

### 2. Mark Entry (Dokumen Valid)
1. Klik tombol "Entry"
2. Konfirmasi "OK"
3. Status berubah → "Sudah Entry"

### 3. Lapor Anomali (Dokumen Error)
1. Klik tombol "Error"
2. Pilih jenis error:
   - Data Tidak Lengkap
   - Format Salah
   - Inkonsistensi Data
3. Isi keterangan detail
4. Klik "Laporkan Error"

## 📋 WORKFLOW
```
Dokumen "Uploaded" → Review → Keputusan:
                                ├─ Valid → "Entry"
                                └─ Error → "Lapor Error"
```

## ⚡ QUICK ACTIONS
- **Proses Dokumen**: Dokumen Survei → Tombol Entry/Error
- **Cek Backlog**: Filter status "Uploaded"
- **Review Error**: Filter status "Error"

## 🚨 ALERT INDICATORS
- Backlog > 50 dokumen: Prioritas processing
- Dokumen error lama: Follow up dengan PCL

## ✅ BEST PRACTICES
1. Konsistensi kriteria quality control
2. Keterangan error yang jelas dan actionable
3. Proses berdasarkan tanggal setor (FIFO)
4. Koordinasi dengan PCL untuk klarifikasi
